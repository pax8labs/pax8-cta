# Issue: Create feature parity test suite for webapp, CLI, and Claude skill

## Problem
AgentSync has three interfaces for the same functionality:
1. **Web App** - UI at localhost:3000
2. **CLI** - `agentsync` command-line tool
3. **Claude Code Skill** - Integration for Claude Code

Currently, there's no automated way to verify these three interfaces provide the same capabilities and consistent behavior. This creates risks:
- Features might be added to one interface but not others
- Error messages could be inconsistent
- API responses might differ
- User experience could be fragmented

## Current Testing Gap

We have individual tests:
- Web: 7 E2E tests (Playwright)
- CLI: 0 tests (Issue #180)
- Claude Skill: 1 E2E test file verifying API integration

But no **cross-interface tests** comparing them.

  let webResult, cliResult, skillResult;

  beforeAll(async () => {
    // Get agents via all three interfaces

    cliResult = await getAgentsViaCLI();
    skillResult = await getAgentsViaSkill();

  it('should return same number of agents', () => {
    expect(webResult.length).toBe(cliResult.length);
    expect(webResult.length).toBe(skillResult.length);
